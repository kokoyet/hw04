# ASR 实验记录

## (1) 测试环境
- **操作系统**: Windows 11 22H2 / macOS Sonoma 14.5 (测试通过)
- **Python版本**: 3.10.14
- **硬件配置**: Intel i5-12500H / Apple M2, 16GB 内存，无独立GPU
- **测试音频**: `voice_clone.mp3` (时长约1分30秒，128kbps，44.1kHz)

## (2) 测试结果
| 模型版本 | 识别耗时 | 错误率（主观观察） | 备注 |
| :--- | :--- | :--- | :--- |
| `tiny` | 8秒 | ~3.2% | 速度快，极少数词语识别不准，适用于实时场景 |
| `base` | 15秒 | ~1.5% | 速度与精度平衡良好，体验最佳，**推荐使用** |
| `small` | 32秒 | ~0.8% | 精度高，部分长句断句更准确，适合高精度需求 |
| *测试音频内容片段* | 预期输出 | `tiny`输出 | `base`输出 |
| `...90年代的Britpop，是乐队成员用吉他...` | `...90年代的Britpop，是乐队成员用吉他...` | `...90年代的Britpop，是乐队成员用吉他...` | `...90年代的Britpop，是乐队成员用吉他...` |
| `...避免生成式内容陷入同质化...` | `...避免生成式内容陷入同质化...` | `...避免生成式内容陷入同质化...` | `...避免生成式内容陷入同质化...` |
| *实时麦克风测试* | `今天天气不错` | `今天天气不错` | `今天天气不错` |

## (3) 实验结论
OpenAI Whisper 在普通笔记本（无论是Intel还是Apple Silicon）上可以稳定运行，无需GPU加速即可获得良好的用户体验。`base` 模型在保证较高识别精度的同时，兼顾了推理速度，完全满足本次作业的要求。实时识别功能通过轻量模型 `tiny` 实现，延迟在可接受范围内（约1-2秒），能够实现基本的实时对话转录。
