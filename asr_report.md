# 任务三：开源语音识别（ASR）调研与实现

## 1. 三种开源 ASR 方案对比

| 对比维度 | OpenAI Whisper | Vosk | FunASR |
| :--- | :--- | :--- | :--- |
| **许可协议** | MIT | Apache 2.0 | MIT |
| **语言与方言支持** | 支持99+语言，中文及部分方言（如粤语）效果好 | 支持20+语言，中文模型多，可扩展 | 专注中文及多方言，支持粤语、四川话等 |
| **模型体量与推理速度** | tiny (~1GB) 到 large-v3 (~10GB)。CPU上tiny极快，large精度高但慢 | 模型小（~50MB），推理速度快，适合低资源设备 | 提供从small到large的模型，速度与精度权衡与Whisper类似 |
| **流式/实时支持** | 原生不支持，需第三方库（如faster-whisper）实现 | 原生支持流式识别，适合实时应用 | 支持流式识别，官方提供实时接口 |
| **依赖与部署难度** | 低。`pip install` 即可，需ffmpeg | 低。Python包安装简单，模型文件需单独下载 | 中。依赖PyTorch等，但官方提供一键安装脚本 |
| **笔记本/PC实测感受** | 操作简单，精度高。`base`模型在CPU上运行流畅，中文识别效果好 | 部署轻量，实时性强，但中文识别精度相比Whisper略低 | 中文精度极高，但首次安装可能较慢，资源消耗与Whisper相当 |

## 2. 选型理由
本项目选择 **OpenAI Whisper** 作为实现方案，主要基于以下几点：

1.  **部署门槛极低**：通过 `pip install openai-whisper` 即可完成核心依赖安装，无需复杂的环境配置，完全符合作业“在笔记本/PC上复现”的要求。
2.  **中文识别精度高**：Whisper 模型在大量多语言数据上训练，对中文普通话的识别效果非常出色，能够很好地处理任务二生成的配音音频。
3.  **模型灵活度高**：提供多种体量的模型（`tiny`, `base`, `small`, `medium`, `large`），可以根据设备的硬件配置（如CPU/GPU、内存）进行灵活选择，在精度和速度之间取得平衡。
4.  **生态与社区支持**：Whisper 是当前最流行的开源ASR方案之一，拥有丰富的社区资源和教程，便于查找问题和获取支持，极大地降低了实现难度。

虽然 Vosk 在实时性上有优势，FunASR 在中文方言上表现更佳，但考虑到本次作业的核心目标是完成一个**可运行、可复现**的识别程序，Whisper 的综合易用性和识别效果使其成为最优选择。
